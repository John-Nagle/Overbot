gpsins

The GPSINS server process that talks to the Novatel GPS Receiver, Xbow AHRS (Attitude Heading Reference System) and Xbow FOG (Fiber Optic Gyro). It also responds to request msgs and responds with a reply that encapsulates all the estimated position, heading and attitude state.

Syntax:

gpsins [option[,option ...]] ... &

Runs on:

X86, QNX Neutrino, needs watchdog timer to be running for communication to happen

Options:

-h | --help Displays help

-V | --version Display version

-v | --verbose Sets verbose flag

-a | --ahrs serial_dev Serial device to use for ahrs

-A | --ahrsLog logFile Filename to log ahrs

-g | --gps serial_dev Serial device to use for gps

-G | --gpsLog logFile Filename to log gps

-f | --fog serial_dev Serial device to use for fog gyro

-F | --fogLog logFile Filename to log fog

-d | --dt time Length of time step

-K | --kfLog logFile Filename to log Kalman Filter

Description:

The GPSINS server talks to the GPS receiver, AHRS, FOG units and the METER server. It combines all the sensor data in Kalman Filters to estimate the state of the vehicle. The internal position update runs at the same rate as the AHRS (60Hz). There are a few threads running within. One that polls on the devices on the serial port. One that waits for requests and responds with the data.

Usage:

gpsins -h

or gpsins --help

will both display help

gpsins -v -g /dev/ser2 -f /dev/ser1

Will set the verbose flag, use /dev/ser2 as the gps serial device and use /dev/ser1 as the fog device, leaving the ahrs device to be its internal default
